# План: Исправление SqliteSaver → AsyncSqliteSaver

## Проблема
`server.py:441` вызывает `p2p_graph.astream()` — асинхронный метод, но `graph.py` использует синхронный `SqliteSaver`, который не поддерживает async операции.

## Анализ
- **SQLite чекпоинтер** — это задокументированное решение для персистентности состояния LangGraph (не ошибка, не заглушка)
- **aiosqlite** уже установлен в `pyproject.toml` 
- Нужно заменить `SqliteSaver` → `AsyncSqliteSaver`

## Сложность
Текущий код создаёт глобальный инстанс `p2p_graph` на уровне модуля (`graph.py:131`). `AsyncSqliteSaver` требует `async with` контекстного менеджера, поэтому нужна ленивая инициализация.

## Решение

### 1. Изменить `app/orchestrator/graph.py`
- Импортировать `AsyncSqliteSaver` из `langgraph.checkpoint.sqlite.aio`
- Создать асинхронную функцию `get_p2p_graph()` с ленивой инициализацией
- Использовать singleton-паттерн для хранения инстанса
- Убрать глобальный `p2p_graph = create_p2p_graph()`

### 2. Изменить `server.py`
- Заменить прямой доступ к `p2p_graph` на `await get_p2p_graph()`
- Обновить все места использования: `astream()`, работа с checkpointer

### 3. Обновить документацию `docs/modules/orchestrator.md`
- Заменить пример с `SqliteSaver` на `AsyncSqliteSaver`

## Файлы для изменения
1. `app/orchestrator/graph.py` — основное изменение
2. `server.py` — адаптация вызовов
3. `docs/modules/orchestrator.md` — обновление документации
