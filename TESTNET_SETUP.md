# Настройка Bybit Testnet для разработки

## Обзор

Bybit Testnet - это полностью изолированная среда для безопасной разработки и тестирования P2P автоматизации. Testnet использует тестовые токены и не затрагивает реальные средства.

## Почему Testnet обязателен

### Безопасность
- ✅ **Нет финансовых рисков** - только тестовые USDT
- ✅ **Изолированная среда** - изменения не влияют на mainnet
- ✅ **Полная функциональность** - все P2P API доступны
- ✅ **Реальные сценарии** - тестирование с настоящими пользователями

### Для разработки
- 🚀 **Быстрое тестирование** - создание ордеров без ожидания
- 🧪 **Полный цикл** - от чата до подтверждения платежа
- 📊 **Реальные данные** - настоящие логи и метрики
- 🔧 **Отладка** - поиск багов в реальной среде

## Шаг 1: Регистрация на Testnet

### 1. Перейдите на testnet.bybit.com
```
https://testnet.bybit.com/
```

### 2. Зарегистрируйтесь
- Используйте **новый email** (не связанный с mainnet аккаунтом)
- Подтвердите email и телефон
- Пройдите KYC верификацию (требуется для P2P)

### 3. Пополните тестовый баланс
- Перейдите в **Assets** → **Spot**
- Используйте **Faucet** для получения тестовых USDT:
  - Нажмите **Faucet** в правом верхнем углу
  - Выберите **USDT** и количество
  - Подтвердите получение

💡 **Совет**: Запрашивайте 100-500 USDT для тестирования

## Шаг 2: Создание API ключей

### 1. Перейдите в API Management
- Войдите в аккаунт
- Откройте **Account & Security** → **API Management**
- Нажмите **Create New Key**

### 2. Настройте права доступа
```
✅ P2P Trading - Read & Write  (ОБЯЗАТЕЛЬНО)
✅ Read                    (ОБЯЗАТЕЛЬНО)
✅ Write                   (рекомендуется)
```
- **НЕ включайте** права на спотовую торговлю или фьючерсы
- **НЕ включайте** права на вывод средств

### 3. Привяжите к IP-адресу (КРИТИЧНО!)
```
🔒 Security Settings:
- Restrict access to trusted IPs only: ✅ ENABLED
- Добавить ваш IP: auto-detect или manual
```
⚠️ **ВАЖНО**: Без IP-привязки API ключ будет работать только 1 час!

### 4. Создайте и сохраните ключи
- **API Key**: `your_api_key_here`
- **API Secret**: `your_api_secret_here` (никогда не показывайте!)
- **Скопируйте** в безопасное место

## Шаг 3: Настройка проекта

### 1. Скопируйте конфигурацию
```bash
cp .env.example .env
```

### 2. Настройте .env файл
```bash
# Bybit Testnet API (ОБЯЗАТЕЛЬНО!)
BYBIT_API_KEY=your_testnet_api_key_here
BYBIT_API_SECRET=your_testnet_api_secret_here
BYBIT_TESTNET=True   # Всегда true для testnet

# Использовать реальный API, не mock
USE_MOCK_DATA=False

# Другие настройки...
```

### 3. Проверьте конфигурацию
```bash
# Валидация при запуске
python -c "import config; config.validate_config()"
```

## Шаг 4: Тестирование интеграции

### 1. Запустите приложение
```bash
# Через Docker
docker-compose up -d

# Или локально
python main.py
```

### 2. Проверьте логи
```
✅ BybitClient initialized with TESTNET API (safe for development)
```

### 3. Тестируйте API
```bash
# Проверить баланс
curl http://127.0.0.1:8000/api/balance

# Проверить объявления
curl http://127.0.0.1:8000/api/ads
```

### 4. Создайте тестовое объявление
1. Откройте [testnet.bybit.com](https://testnet.bybit.com/)
2. Перейдите в **P2P Trading**
3. Создайте объявление на продажу USDT
4. Запустите мониторинг в веб-интерфейсе

## Шаг 5: Безопасная разработка

### Правила работы с testnet
- ✅ **Всегда проверяйте** `BYBIT_TESTNET=True`
- ✅ **Регулярно обновляйте** API ключи (ежемесячно)
- ✅ **Мониторьте логи** на предмет ошибок API
- ✅ **Используйте разные аккаунты** для разных тестов

### Избегайте распространенных ошибок
- ❌ **Не используйте mainnet ключи** в testnet режиме
- ❌ **Не коммитьте .env** с реальными ключами
- ❌ **Не тестируйте** на mainnet случайно
- ❌ **Не используйте** одни ключи для нескольких проектов

### Мониторинг использования

```bash
curl https://api-testnet.bybit.com/v5/api-key -H "X-BAPI-API-KEY: your_key"

# Проверить rate limits
# В логах приложения ищите rate limit предупреждения
```

## Troubleshooting

### Проблема: API ключ не работает
```
❌ Failed to initialize Bybit API client
```
**Решение:**
1. Проверьте корректность API ключа и секрета
2. Убедитесь в IP-привязке
3. Проверьте, что аккаунт верифицирован
4. Попробуйте пересоздать API ключ

### Проблема: Rate limit
```
❌ Too many requests
```
**Решение:**
1. Уменьшите частоту запросов
2. Добавьте задержки между вызовами
3. Используйте кэширование для повторяющихся данных

### Проблема: Нет объявлений
```
{"success": true, "data": []}
```
**Решение:**
1. Создайте объявление на testnet.bybit.com
2. Проверьте, что объявление активно
3. Подождите несколько минут для синхронизации

### Проблема: Ошибка аутентификации
```
❌ Invalid API key
```
**Решение:**
1. Проверьте корректность ключа в .env
2. Убедитесь, что используете testnet ключи
3. Проверьте срок действия ключа
