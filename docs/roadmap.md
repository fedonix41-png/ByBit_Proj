# План развития проекта (Roadmap)

В этом документе зафиксированы приоритетные задачи и вектор развития Умного P2P-Ассистента на ближайшие релизы.

## Приоритет 1: Завершение интеграции Bybit ↔ Zavod
- [ ] **Клиент Zavod:** Реализация полноценного взаимодействия с платформой Zavod (парсинг XHR-запросов Vue.js приложения или внедрение Playwright для эмуляции браузера).
- [ ] **UI Оператора:** Добавление в веб-интерфейс (FastAPI) кнопок быстрой передачи извлеченных ИИ реквизитов на создание заявки в Zavod.
- [ ] **Сквозной флоу:** Отладка цепочки: Ордер Bybit -> Сбор реквизитов -> Создание на Zavod -> Получение чека -> Отправка в Bybit.

## Приоритет 2: Обучение и Точность ИИ
- [ ] **Шаблоны диалогов:** Сбор и внедрение эталонных диалогов операторов в `response_templates.yaml` для повышения качества ответов `ResponseGenerator`.
- [ ] **Тюнинг OCR (PaymentParser):** Адаптация и тестирование инструмента под специфические чеки, выдаваемые платформой Zavod, для автоматической валидации оплат.
- [ ] **Скрипты FraudAnalyzer:** Обучение системы выявлять подозрительные паттерны поведения в чате на основе исторических данных мошеннических сделок.

## Приоритет 3: Инфраструктура и Удобство
- [ ] **Метрики и Мониторинг:** Развертывание Prometheus/Grafana для отслеживания процента успешных автоматизаций и скорости отклика AI-агентов.
- [ ] **Админ-панель:** Создание дашборда для управления шаблонами ответов и правилами фрод-анализатора без правки кода.
- [ ] **CI/CD:** Настройка автоматического тестирования и деплоя (GitHub Actions).

*(Примечание: завершенные этапы и история изменений ведутся в логах коммитов и `status.md`, данный файл предназначен исключительно для планирования)*
