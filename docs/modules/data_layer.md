# Слой Данных (База данных и Безопасность)

**Расположение:** `app/database/`, `app/core/`

Этот модуль описывает структуру хранения данных, миграции и базовые политики безопасности (Rate Limit, Audit Logging), которые опираются на БД.

## 1. Технологический стек
- **СУБД:** PostgreSQL (через Docker Compose)
- **ORM:** SQLAlchemy (в `app/database/models.py`)
- **Миграции:** Alembic (в `alembic/versions/`)
- **Кэш/Rate Limit:** Redis

## 2. Основные Модели (`models.py`)

- **Order:** Хранение P2P ордеров (ID, валюта, статус, контрагент).
- **Message:** Сообщения чата, их AI-анализ (намерения) и оценки уверенности.
- **Decision:** Человеческие и автоматические решения (одобрения риска, действия).
- **Transaction:** Финансовые транзакции и их статус (ожидание чека, завершено).
- **AIInteraction:** История обращений к LLM (провайдер, кол-во токенов, стоимость, latency).
- **SecurityAuditLog:** Аудит безопасности (входы, блокировки, refresh токенов).
- **ViolationHistory:** История нарушений пользователей (спам-фильтр, блэклисты).

## 3. Миграции (Alembic)

```bash
# Применить миграции
uv run alembic upgrade head

# Создать новую миграцию после изменения models.py
uv run alembic revision --autogenerate -m "описание"
```

## 4. Резервное копирование

Бэкапы создаются автоматически через скрипт `scripts/backup_db.sh`.
Крон запускается ежедневно (в 2:00) внутри контейнера.
Файлы сохраняются в директории `./backups` (volume).

## 5. Защита и Логирование
Все API-эндпоинты, работающие с данными, обёрнуты в:
- **JWT Auth** (access и refresh токены).
- **Rate Limit** (Sliding window через Redis).
- **CORS** (Whitelist) и **Security Headers**.
Настройка данных параметров осуществляется через `app/core/security_headers.py` и `app/core/auth.py`.
