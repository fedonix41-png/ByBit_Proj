# AI-движок и Оркестратор

**Расположение:** `app/ai_agents/`, `app/orchestrator/`

Этот модуль описывает ядро принятия решений (оркестратор на базе LangGraph) и набор специализированных AI-агентов, которые извлекают данные и формируют ответы. Роль ИИ — **Ассистент**, оператор всегда принимает итоговое решение (Human-in-the-Loop).

## 1. Оркестратор (LangGraph)

Оркестратор собирает вызовы к Bybit, вызовы к AI и логику ожиданий в единый направленный граф.

**Ключевые файлы:**
- `state.py` — `P2PAutomationState` (содержит данные ордера, переписку, скоринг фрода и статус).
- `graph.py` — Сборка 12-узлового графа (fetch_order -> check_messages -> classify_intent -> ... -> submit_processing).
- `nodes.py` — Асинхронные функции (узлы), где происходит вызов AI или API.
- `edges.py` — Условные переходы (например, если чек получен -> парсим, иначе -> ждём).

**Точки прерывания (Human-in-the-Loop):**
- `await_response_approval` — Одобрение текста ответа перед отправкой в Bybit.
- `await_risk_approval` — Одобрение реквизитов (и оценка риска) перед созданием заявки на Zavod.

## 2. AI-агенты

Все агенты наследуются от `BaseAIAgent`, поддерживающего OpenAI, Anthropic, Groq, OpenRouter, и локальные модели.

| Агент | Назначение | Узел графа |
|-------|------------|------------|
| `IntentClassifier` | Понимание контекста сообщения клиента (9 типов намерений). | `classify_intent` |
| `ResponseGenerator` | Формирование ответа по шаблонам (`response_templates.yaml`). | `generate_response` |
| `FraudAnalyzer` | Анализ реквизитов из чата по базе BIN-кодов и правилам (rule-based + AI). | `analyze_risk` |
| `PaymentParser` | OCR-парсинг чеков от платформы Zavod для верификации оплаты. | `parse_payment` |
| `AILogger` | Сохранение логов вызовов LLM с подсчётом стоимости токенов. | - |

## 3. Интеграция с моделями

Мы используем `OpenRouterClient` (адаптер) для доступа к топовым LLM (GPT-4o, Claude) и для Vision-задач. Выбор провайдера гибкий, переключается в настройках, но основной поток завязан на семантику и `BaseAIAgent.generate()`.
