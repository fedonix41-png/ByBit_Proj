# Текущий статус проекта (реальное состояние)

Последнее обновление: 2026-04-29

## ✅ Реализовано

### Ядро
- **LangGraph оркестратор v2** (`app/orchestrator/`): **единственный граф** с 12 узлами, поддержка прерываний (response approval, risk approval), сохранение состояния в SQLite через `SqliteSaver`. Старый корневой `graph.py` удалён.
- **AI-агенты** (`app/ai_agents/`): мульти-провайдерная генерация (OpenAI, Anthropic, Groq, Together, Mistral, локальный), мок-режим. Реализованы: `IntentClassifier`, `ResponseGenerator`, `PaymentParser`, `FraudAnalyzer`. Все используют `BaseAIAgent`.
- **Telegram-бот** (`app/infrastructure/interface/bot.py`): команды, обработка текста, голоса, фото. Интегрирован с оркестратором через `p2p_bridge`.
- **Bybit-клиент** (`bybit_client.py` в корне): поддержка testnet API (если установлен пакет `bybit-p2p`), мок-фолбек. Методы: получение ордеров, чата, баланса, подтверждение платежа, создание/отмена ордеров.
- **FastAPI веб-интерфейс** (`server.py`): REST API для ордеров, баланса, истории, мониторинга; WebSocket для real-time обновлений. **Переключен на продвинутый оркестратор.**
- **База данных**: PostgreSQL через SQLAlchemy, модели Order, Message, Decision, Transaction, AIInteraction; миграции Alembic.
- **Docker-инфраструктура**: `docker-compose.yml` (PostgreSQL, app, telegram_bot), Dockerfile.

### Что именно работает
- Telegram-бот и веб-интерфейс используют **один граф** (`app/orchestrator/graph.py`).
- Telegram-бот принимает сообщения, запускает граф, классифицирует интенты (через AI), генерирует ответ, останавливается для подтверждения.  
- Администратор может подтвердить/отклонить ответ через веб-интерфейс (REST API).  
- При подтверждении оплаты (пока в мок-режиме) граф продолжается до завершения.  
- Bybit-клиент в тестовой среде получает реальные данные, но fallback к моку при ошибках.
- В веб-интерфейсе видны объявления, чат, баланс (в мок-режиме — тестовые данные).

## ⚠️ Частично реализовано / в процессе
- **PaymentParser (OCR)**: базовая интеграция с pytesseract; качество распознавания не верифицировано.
- **FraudAnalyzer**: комбинация правил и AI, но правила минимальны.
- **Интеграция с Processing API**: только мок-заглушка в `processing_client.py`.
